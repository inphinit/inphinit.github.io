Controllers

Controllers are one of the different ways to organize your project, with controllers you can centralize the request handling logic.

Create routes

Para criar um Controller crie um arquivo PHP na pasta `system/Controllers/`, por exemplo: `FooBar.php`, então adicione o seguinte conteudo:

<?php namespace Controllers; class FooBar { public function hello($app, $params) { return 'Hello World'; } public function hundop($app, $params) { return 'Hundo P!'; } }

Então no `main.php` crie uma rota apontando para o controller

<?php $app->action('GET', '/hello', 'FooBar::hello'); $app->action('GET', '/100', 'FooBar::hundop');

Também é possivel adicionar em um subdiretório, por exemplo `system/Controllers/Foo/Bar/Baz.php`, e adicione isso ao conteudo (não esqueça de ajustar o namespace):

<?php namespace Controllers\Foo\Bar; class Baz { public function say($app, $params) { return 'Hello World'; } }

Então adicione a rota:

<?php $app->action('GET', '/say', 'Controllers\Foo\Bar\Baz::say');

Prefixando rotas

Para evitar escrever repetidamente acaso tenha um namespace longo, você pode mudar o prefixo do namespace para as rotas seguintes, ou rotas por escopo, por exemplo:

Note: The namespace prefix in `dev.php` will not be propagated to `main.php`; in both scripts, the initial value will always be `Controllers`.

<?php $app->setNamespace('Controllers\\Foo\\Bar'); // system/Controllers/Foo/Bar/Baz.php $app->action('GET', '/', 'Baz::index'); $app->action('GET', '/create', 'Baz::create'); $app->action('GET', '/read', 'Baz::read'); $app->action('GET', '/update', 'Baz::update'); $app->action('GET', '/delete', 'Baz::delete'); // system/Controllers/Foo/Bar/Baz.php $app->setNamespace('Controllers\\Halloween\\Ghost\\'); // system/Controllers/Halloween/Ghost/Boo.php $app->action('GET', '/halloween/trick', 'Boo::trick'); $app->action('GET', '/halloween/treat', 'Boo::treat');

A pasta Controllers não é obrigatória, você pode mudar completamente o namespace, mas para isso será preciso adicionar a entrada ao `"psr-4":` no `composer.json`, que atualmente deve ser:

... "autoload": { "psr-4": { "Commands\\": "system/Commands", "Controllers\\": "system/Controllers", "Models\\": "system/Models", } }, ...

Tornando em algo como:

... "autoload": { "psr-4": { "Commands\\": "system/Commands", "Controllers\\": "system/Controllers", "CustomFolder\\": "system/CustomFolder", "Models\\": "system/Models", } }, ...

Então configure no main.php:

<?php $app->setNamespace('CustomFolder\\Foo\\Bar'); // system/CustomFolder/Foo/Bar/Baz.php $app->action('GET', '/', 'Baz::index'); $app->action('GET', '/create', 'Baz::create'); $app->action('GET', '/read', 'Baz::read'); $app->action('GET', '/update', 'Baz::update'); $app->action('GET', '/delete', 'Baz::delete'); // Restaura o prefixo original $app->setNamespace('Controllers'); ...
